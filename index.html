API Documentation

Routes

  • GET /
    You are here.
  • 🔐 GET /users
    Returns all users.
  • 🔐 GET /users/{specifier}
    Returns a user with the corresponding specifier (either an id or username) if it exists. Sadly won't work by username if you're called "me".
  • 🔐 GET /users/me
    Returns information stored in your token.
  • POST /sign-up
    Creates a new user with the specified username and password if the username is not already in use. Don't worry, the password is hashed.
  • POST /sign-in
    Returns a JWT that is used as bearer token in secure routes requiring authentication.
  • 🔐 PUT /change-username
    Changes your username to a new one if the new one is not already in use.

Notes

  • Routes with the 🔐 requires authentication.
